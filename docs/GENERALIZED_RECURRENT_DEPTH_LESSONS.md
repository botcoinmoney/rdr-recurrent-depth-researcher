# Generalized Recurrent-Depth Lessons

These lessons are now treated as first-class harness constraints, not optional notes.

## Imported From Prior BOTCOIN Evaluations

1. Compare every candidate against a no-adapter baseline depth curve.
2. Track absolute deep-depth performance, not only slope.
3. Require a matched scramble control before claiming data-specific latent gains.
4. Treat evidence availability as part of the mechanism.
5. Evaluate latent metrics and surface behavior separately.
6. Sweep multiple recurrence depths before rejecting or accepting a method.

## OpenMythos-Inspired Knob Families

`OpenMythos` is a speculative open-source reconstruction, not an Anthropic primary source. It is useful here only as a source of hypothesis families to test dynamically:

- loop-depth scheduling
- depth extrapolation beyond train-time loops
- frozen-input or evidence reinjection
- spectral-stability / LTI-style recurrence constraints
- ACT-style adaptive halting
- loop-index signals
- per-loop adapter rank
- dense vs sparse recurrent FFN structure
- attention/cache mode such as standard, GQA, or MLA
- prelude / recurrent / coda partitioning

None of these should be treated as default truths. The harness surfaces them as reusable knobs that must still pass the control stack above.
